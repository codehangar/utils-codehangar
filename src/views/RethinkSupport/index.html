Getting Started

Setup

To get started you will need to download the app. Once downloaded and installed simply launch the application! When you launch the application you will be greeted with the screen below.

Connections

ReQL Pro gives you a way to manage multiple database connections. In this section lets go over adding, editing, and deleting connections.

Add Connection

Click the "+" button in the bottom left corner of the application. A form will appear (as shown on right) with fields for name, host, and port. Upon filling out these form elements just click the add connection button. Success! You added your first RethinkDB connection.

Edit Connection

Now that you have a connection setup we can edit details of that connection whenever we need too. Click the pencil icon to the right of the name of your connection in the center column. Make any edits nesseccary then click the "Edit Connection" button.

Toggle Connections

Once you have multiple connections setup you can quickly toggle between them by clicking on them in the very left column in the app.

Manage Data

Okay lets manage some data! In this section we will go over inserting/updating/deleting records, toggling between data visualizations, and using pagination.

Inserting Records

To insert records into a table have a table selected from the middle column of the application. Then in the left top navigation click the "+" button. This will give you a text editor where you can construct JSON objects to insert into the table.

Updating Records

On the table view layout you can use the pencil icon to update an existing record. You will find the update view interface familiar if you have inserted a record before.

Deleting Records

On the tableview layout. You can use the garbage can icon to remove the record.

Toggling data visualizations

The top row of the right column contains two icons to the very right. The first one to the left is a tree, this will give you a JSON Tree visulation for your data. The second icon looks like a table, this icon will give you a datatable representation.

Tree View

The tree views main benefit is for exploring deeply nested records.

Datatable View

The datatable view has multiple benefits. This view gives you the action buttons to edit and delete records. It also allows for quick inline edits of certain field types. It also provides a traditional database feel.
